## 1과목: 데이터 통신

---

### 1.1 전송 매체

| 매체 | 종류 | 특징 |
|------|------|------|
| 꼬임쌍선 (UTP/STP) | 유선 | 가장 저렴. Cat5e(1Gbps), Cat6(10Gbps). 전자기 간섭에 약함 |
| 동축 케이블 | 유선 | UTP보다 대역폭 높음. 케이블 TV에 사용 |
| 광섬유 | 유선 | 가장 빠르고 장거리 전송. 전자기 간섭 없음. 단일모드/다중모드 |
| 적외선 | 무선 | 직진성. 장애물 통과 불가. 리모컨 |
| 라디오파 | 무선 | 장애물 통과 가능. Wi-Fi, 블루투스 |
| 마이크로파 | 무선 | 직진성. 중계기 필요. 위성 통신 |

#### 단일모드 vs 다중모드 광섬유
- **단일모드(Single-mode)**: 코어 직경 작음(8~10μm). 장거리 전송. 레이저 광원
- **다중모드(Multi-mode)**: 코어 직경 큼(50~62.5μm). 단거리 전송. LED 광원

### 1.2 흐름 제어 (Flow Control)

수신 측이 송신 측의 전송 속도를 조절하는 기법

| 기법 | 설명 |
|------|------|
| **정지-대기 (Stop-and-Wait)** | 한 프레임 전송 후 ACK를 기다림. 단순하지만 비효율적 |
| **슬라이딩 윈도우** | 윈도우 크기만큼 연속 전송 가능. ACK 없이도 여러 프레임 전송 |
| **XON/XOFF** | 소프트웨어적 흐름 제어. XON(전송 재개), XOFF(전송 중지) |

#### 슬라이딩 윈도우 계산
- 윈도우 크기 W, 순서번호 비트 수 n
- **Go-Back-N**: W ≤ 2ⁿ - 1
- **Selective Repeat**: W ≤ 2ⁿ⁻¹

### 1.3 다중 접속 프로토콜 (MAC)

| 분류 | 프로토콜 | 설명 |
|------|---------|------|
| 경쟁 방식 | **ALOHA** | 충돌 시 랜덤 시간 후 재전송. Pure(최대 18.4%) / Slotted(최대 36.8%) |
| 경쟁 방식 | **CSMA** | 전송 전 채널 감지. 1-persistent/non-persistent/p-persistent |
| 경쟁 방식 | **CSMA/CD** | 유선 이더넷. 충돌 감지 후 중단, 재전송 |
| 경쟁 방식 | **CSMA/CA** | 무선 Wi-Fi. 충돌 회피. RTS/CTS 사용 |
| 비경쟁 방식 | **토큰 링** | 토큰을 가진 스테이션만 전송 가능 |
| 비경쟁 방식 | **토큰 버스** | 버스 토폴로지에서 토큰 전달 |
| 비경쟁 방식 | **폴링** | 중앙 제어 장치가 순서대로 전송 허가 |

#### CSMA/CD vs CSMA/CA
| 구분 | CSMA/CD | CSMA/CA |
|------|---------|---------|
| 환경 | 유선 (이더넷) | 무선 (Wi-Fi) |
| 방식 | 충돌 **감지** (Detection) | 충돌 **회피** (Avoidance) |
| 이유 | 유선은 충돌 감지 가능 | 무선은 숨은 터미널 문제로 감지 어려움 |
| 표준 | IEEE 802.3 | IEEE 802.11 |

### 1.4 데이터 부호화

| 부호화 | 특징 |
|--------|------|
| NRZ-L | 0=양, 1=음. 동기화 어려움 |
| NRZ-I | 1일 때 신호 전이. 연속 0에서 동기화 어려움 |
| 맨체스터 | 비트 중앙에서 항상 전이. 이더넷 사용. 대역폭 2배 필요 |
| 차동 맨체스터 | 비트 시작에서 전이 유무로 구분. 토큰 링 사용 |
| AMI | 0=0전압, 1=교대 양/음. 연속 0에서 동기화 어려움 |
| B8ZS | AMI 개선. 연속 8개 0을 특수 패턴으로 대체 |
| HDB3 | AMI 개선. 연속 4개 0을 특수 패턴으로 대체 |

### 1.5 변조 방식

| 변조 | 변화시키는 요소 | 디지털→아날로그 |
|------|--------------|--------------|
| ASK | 진폭 (Amplitude) | 0과 1을 진폭 차이로 구분 |
| FSK | 주파수 (Frequency) | 0과 1을 주파수 차이로 구분 |
| PSK | 위상 (Phase) | 0과 1을 위상 차이로 구분 |
| QAM | 진폭 + 위상 | ASK+PSK 결합. 고속 전송에 사용 |

### [데이터 통신 예상 문제]

**문제 1.** 이더넷(IEEE 802.3)에서 사용하는 다중 접속 방식은?
1. CSMA/CA
2. CSMA/CD
3. 토큰 패싱
4. 폴링

> **정답: 2번**. 이더넷은 CSMA/CD를 사용합니다. CSMA/CA는 무선 LAN(802.11)에서 사용합니다.

**문제 2.** 슬라이딩 윈도우 프로토콜에서 Go-Back-N 방식을 사용하고, 순서번호 비트 수가 3비트일 때, 최대 윈도우 크기는?
1. 3
2. 4
3. 7
4. 8

> **정답: 3번**. Go-Back-N의 최대 윈도우 크기 = 2ⁿ - 1 = 2³ - 1 = 7

**문제 3.** 다음 중 광섬유 케이블의 특징으로 틀린 것은?
1. 전자기 간섭(EMI)에 강하다
2. 도청이 어렵다
3. 동축 케이블에 비해 대역폭이 좁다
4. 장거리 전송에 유리하다

> **정답: 3번**. 광섬유는 모든 유선 매체 중 가장 넓은 대역폭을 제공합니다.

**문제 4.** 디지털 데이터를 아날로그 신호로 변환할 때, 진폭과 위상을 동시에 변화시키는 변조 방식은?
1. ASK
2. FSK
3. PSK
4. QAM

> **정답: 4번**. QAM(Quadrature Amplitude Modulation)은 ASK와 PSK를 결합합니다.

**문제 5.** 맨체스터 부호화에 대한 설명으로 옳은 것은?
1. 비트 구간의 시작에서만 전이가 발생한다
2. 비트 구간의 중앙에서 항상 전이가 발생하여 자체 동기화가 가능하다
3. NRZ 방식보다 필요한 대역폭이 작다
4. 토큰 링 네트워크에서 사용된다

> **정답: 2번**. 맨체스터 부호화는 비트 중앙에서 항상 전이가 발생하여 클럭 동기화가 가능합니다. 이더넷(802.3)에서 사용하며, 대역폭은 NRZ의 2배가 필요합니다. 토큰 링은 차동 맨체스터를 사용합니다.

### 2.1 IP 서브넷팅

#### 서브넷 마스크 계산
- IP 주소: 네트워크 부분 + 호스트 부분
- 서브넷 마스크의 1비트 = 네트워크, 0비트 = 호스트
- 사용 가능한 호스트 수 = 2^(호스트 비트 수) - 2 (네트워크 주소, 브로드캐스트 주소 제외)

**예시**: 192.168.1.0/26
- /26 → 서브넷 마스크: 255.255.255.192
- 호스트 비트: 32 - 26 = 6비트
- 서브넷당 호스트 수: 2⁶ - 2 = 62개
- 서브넷 수: 2² = 4개 (마지막 옥텟에서 2비트를 빌림)
- 서브넷 범위: .0~.63, .64~.127, .128~.191, .192~.255

### 2.2 라우팅 프로토콜

| 분류 | 프로토콜 | 알고리즘 | 특징 |
|------|---------|---------|------|
| IGP | **RIP** | 거리 벡터 | 홉 수 기반. 최대 15홉. 30초마다 업데이트. 소규모 |
| IGP | **OSPF** | 링크 상태 | 대역폭 기반. 영역(Area) 개념. 대규모. 다익스트라 사용 |
| IGP | **EIGRP** | 하이브리드 | Cisco 전용. DUAL 알고리즘 |
| EGP | **BGP** | 경로 벡터 | AS 간 라우팅. 인터넷 백본. TCP 179번 포트 |

#### 거리 벡터 vs 링크 상태
| 구분 | 거리 벡터 | 링크 상태 |
|------|----------|----------|
| 정보 교환 | 이웃에게 전체 라우팅 테이블 | 모든 라우터에 링크 상태 정보 |
| 수렴 속도 | 느림 | 빠름 |
| 대역폭 사용 | 주기적으로 많음 | 변경 시에만 |
| 문제점 | 카운트 투 인피니티 | 계산 복잡 |
| 알고리즘 | 벨만-포드 | 다익스트라 |

### 2.3 ARP / RARP / DHCP / DNS

| 프로토콜 | 기능 | 방향 |
|----------|------|------|
| **ARP** | IP 주소 → MAC 주소 변환 | 논리 → 물리 |
| **RARP** | MAC 주소 → IP 주소 변환 | 물리 → 논리 |
| **DHCP** | IP 주소 자동 할당 | 서버가 클라이언트에 IP 배정 |
| **DNS** | 도메인 이름 → IP 주소 변환 | 이름 해석 |

#### ARP 동작 과정
1. ARP 요청(Request): 브로드캐스트로 "이 IP의 MAC 주소 알려주세요"
2. ARP 응답(Reply): 해당 호스트가 유니캐스트로 자신의 MAC 주소 응답
3. ARP 캐시에 저장하여 재사용

#### DHCP 동작 과정 (DORA)
1. **Discover**: 클라이언트가 브로드캐스트로 DHCP 서버 탐색
2. **Offer**: 서버가 사용 가능한 IP 주소를 제안
3. **Request**: 클라이언트가 제안된 IP 요청
4. **Acknowledge**: 서버가 IP 할당 확인

### 2.4 VLAN / NAT / 방화벽

#### VLAN (Virtual LAN)
- 물리적 위치와 관계없이 논리적으로 LAN을 분리
- 브로드캐스트 도메인을 나눔
- 보안 향상, 트래픽 관리 용이
- IEEE 802.1Q 태깅 방식

#### NAT (Network Address Translation)
- 사설 IP를 공인 IP로 변환
- IPv4 주소 부족 문제 해결
- 내부 네트워크 보안 향상
- 종류: 정적 NAT, 동적 NAT, PAT(포트 기반)

#### 방화벽 종류
| 종류 | 동작 계층 | 특징 |
|------|----------|------|
| 패킷 필터링 | 3~4계층 | IP, 포트 기반 필터링. 빠르지만 내용 검사 불가 |
| 상태 검사 (Stateful) | 3~4계층 | 연결 상태를 추적하여 필터링 |
| 애플리케이션 게이트웨이 (프록시) | 7계층 | 내용까지 검사. 느리지만 보안성 높음 |
| 차세대 방화벽 (NGFW) | 전 계층 | DPI + IPS + 앱 인식 통합 |

### 2.5 IPv4 vs IPv6

| 구분 | IPv4 | IPv6 |
|------|------|------|
| 주소 길이 | 32비트 | 128비트 |
| 주소 표기 | 10진수 (192.168.1.1) | 16진수 (2001:db8::1) |
| 주소 수 | 약 43억 개 | 사실상 무한 |
| 헤더 | 가변 길이, 복잡 | 고정 40바이트, 단순 |
| 체크섬 | 있음 | 없음 (상위 계층에 맡김) |
| 브로드캐스트 | 있음 | 없음 (멀티캐스트로 대체) |
| IPSec | 선택 | 기본 내장 |
| 자동 설정 | DHCP 필요 | SLAAC로 자동 설정 가능 |

#### IPv4에서 IPv6 전환 기술
| 기술 | 설명 |
|------|------|
| 듀얼 스택 | IPv4와 IPv6 동시 지원 |
| 터널링 | IPv6 패킷을 IPv4 패킷 안에 캡슐화 |
| 주소 변환(NAT64) | IPv4와 IPv6 간 주소 변환 |

### 2.7 TCP 3-Way Handshake / 4-Way Handshake

#### 연결 수립 (3-Way Handshake)
```
클라이언트 → SYN → 서버         (연결 요청)
클라이언트 ← SYN+ACK ← 서버    (요청 수락)
클라이언트 → ACK → 서버         (연결 확립)
```

#### 연결 해제 (4-Way Handshake)
```
클라이언트 → FIN → 서버         (연결 종료 요청)
클라이언트 ← ACK ← 서버        (요청 확인)
클라이언트 ← FIN ← 서버        (서버도 종료 준비)
클라이언트 → ACK → 서버         (종료 확인)
```

### 2.8 TCP 혼잡 제어

| 기법 | 설명 |
|------|------|
| **느린 시작 (Slow Start)** | 윈도우 크기를 1에서 시작해 지수적으로 증가 (1→2→4→8...) |
| **혼잡 회피 (Congestion Avoidance)** | 임계값 이후 선형적으로 증가 (1씩) |
| **빠른 재전송 (Fast Retransmit)** | 3개의 중복 ACK 수신 시 즉시 재전송 |
| **빠른 회복 (Fast Recovery)** | 빠른 재전송 후 느린 시작 대신 혼잡 회피 단계로 진입 |

### [네트워크 예상 문제]

**문제 1.** 192.168.10.0/28 네트워크에서 사용 가능한 호스트 주소의 수는?
1. 14
2. 16
3. 28
4. 30

> **정답: 1번**. /28이면 호스트 비트 = 32 - 28 = 4비트. 호스트 수 = 2⁴ - 2 = 14개

**문제 2.** 다음 중 링크 상태(Link-State) 라우팅 프로토콜은?
1. RIP
2. OSPF
3. BGP
4. EIGRP

> **정답: 2번**. OSPF는 링크 상태 알고리즘을 사용하며, 다익스트라 알고리즘으로 최단 경로를 계산합니다.

**문제 3.** IP 주소를 MAC 주소로 변환하는 프로토콜은?
1. RARP
2. ARP
3. DHCP
4. DNS

> **정답: 2번**. ARP는 IP→MAC 변환. RARP는 MAC→IP 변환입니다.

**문제 4.** TCP 3-Way Handshake에서 연결 수립 과정의 올바른 순서는?
1. SYN → ACK → SYN+ACK
2. SYN → SYN+ACK → ACK
3. ACK → SYN → SYN+ACK
4. SYN+ACK → SYN → ACK

> **정답: 2번**

**문제 5.** IPv6에 대한 설명으로 틀린 것은?
1. 128비트 주소 체계를 사용한다
2. 헤더에 체크섬 필드가 포함되어 있다
3. 브로드캐스트가 없고 멀티캐스트를 사용한다
4. IPSec이 기본 내장되어 있다

> **정답: 2번**. IPv6는 헤더를 단순화하기 위해 체크섬 필드를 제거했습니다.

**문제 6.** TCP의 혼잡 제어에서, 3개의 중복 ACK를 수신했을 때 수행하는 동작은?
1. 느린 시작
2. 혼잡 회피
3. 빠른 재전송
4. 연결 종료

> **정답: 3번**. 3개의 중복 ACK 수신 시 타임아웃을 기다리지 않고 즉시 재전송합니다.

**문제 7.** NAT의 주요 목적으로 가장 적절한 것은?
1. 데이터 암호화
2. 라우팅 성능 향상
3. 사설 IP와 공인 IP 간 주소 변환
4. DNS 서비스 제공

> **정답: 3번**

### 3.1 프로세스 상태 전이

```
생성(New) → 준비(Ready) → 실행(Running) → 종료(Terminated)
                 ↑              ↓
                 └── 대기(Waiting) ──┘
```

| 전이 | 설명 | 발생 원인 |
|------|------|----------|
| 생성 → 준비 | 승인(Admit) | 프로세스 생성 완료 |
| 준비 → 실행 | 디스패치(Dispatch) | 스케줄러가 CPU 할당 |
| 실행 → 준비 | 타임아웃/선점 | 시간 할당량 만료, 높은 우선순위 프로세스 도착 |
| 실행 → 대기 | 입출력 요청 | I/O 또는 이벤트 대기 |
| 대기 → 준비 | 입출력 완료 | I/O 완료 인터럽트 |
| 실행 → 종료 | 완료/오류 | 프로세스 정상 종료 또는 오류 |

### 3.2 프로세스 vs 스레드

| 구분 | 프로세스 | 스레드 |
|------|---------|--------|
| 정의 | 실행 중인 프로그램의 인스턴스 | 프로세스 내의 실행 단위 |
| 메모리 | 독립적 주소 공간 | 프로세스의 메모리 공유 (코드, 데이터, 힙) |
| 고유 영역 | 전체 (코드+데이터+스택+힙) | 스택, PC, 레지스터만 고유 |
| 생성 비용 | 큼 (fork) | 작음 |
| 통신 | IPC 필요 (파이프, 소켓 등) | 공유 메모리로 직접 통신 |
| 문맥 교환 | 오버헤드 큼 | 오버헤드 작음 |

#### 사용자 수준 스레드 vs 커널 수준 스레드
| 구분 | 사용자 수준 | 커널 수준 |
|------|-----------|----------|
| 관리 주체 | 사용자 공간의 라이브러리 | 운영체제 커널 |
| 문맥 교환 | 빠름 (커널 개입 없음) | 느림 (커널 모드 전환) |
| 스케줄링 | 프로세스 단위 | 스레드 단위 |
| 블로킹 I/O | 전체 프로세스 블록 | 해당 스레드만 블록 |
| 다중 CPU | 활용 불가 | 활용 가능 |

### 3.3 IPC (프로세스 간 통신)

| 방식 | 설명 |
|------|------|
| **파이프 (Pipe)** | 단방향 통신. 부모-자식 프로세스 간 사용 |
| **명명된 파이프 (Named Pipe/FIFO)** | 이름이 있는 파이프. 관련 없는 프로세스 간 사용 가능 |
| **메시지 큐** | 커널이 관리하는 큐를 통해 메시지 전달 |
| **공유 메모리** | 가장 빠른 IPC. 같은 메모리 영역을 공유. 동기화 필요 |
| **소켓** | 네트워크 통신 가능. 양방향. 서로 다른 시스템 간 통신 |
| **시그널** | 비동기 이벤트 통보. 인터럽트와 유사 |
| **세마포어** | 동기화 도구. 공유 자원 접근 제어 |

### 3.4 메모리 관리

#### 연속 할당 기법
| 기법 | 설명 | 장단점 |
|------|------|--------|
| **최초 적합 (First Fit)** | 첫 번째로 충분한 공간에 할당 | 빠름, 앞쪽에 단편화 |
| **최적 적합 (Best Fit)** | 가장 작은 충분한 공간에 할당 | 공간 효율적, 작은 조각 많이 발생 |
| **최악 적합 (Worst Fit)** | 가장 큰 공간에 할당 | 남은 공간이 큼, 큰 프로세스에 불리 |

#### 단편화
| 종류 | 설명 | 해결 방법 |
|------|------|----------|
| **외부 단편화** | 충분한 총 메모리가 있지만 연속 공간이 없음 | 압축(Compaction), 페이징 |
| **내부 단편화** | 할당된 블록 내 사용되지 않는 공간 | 세그멘테이션 |

#### 페이징 vs 세그멘테이션
| 구분 | 페이징 | 세그멘테이션 |
|------|--------|------------|
| 분할 단위 | 고정 크기 (페이지) | 가변 크기 (세그먼트) |
| 외부 단편화 | 없음 | 발생 가능 |
| 내부 단편화 | 발생 가능 | 없음 |
| 주소 변환 | 페이지 테이블 | 세그먼트 테이블 |
| 논리적 의미 | 없음 (물리적 분할) | 있음 (코드, 데이터, 스택 등) |

### 3.5 파일 시스템

#### 파일 할당 방법
| 방법 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **연속 할당** | 연속된 블록에 저장 | 순차/직접 접근 빠름 | 외부 단편화, 파일 크기 예측 필요 |
| **연결 할당** | 각 블록이 다음 블록 포인터 보유 | 외부 단편화 없음 | 직접 접근 느림, 포인터 손실 위험 |
| **인덱스 할당** | 인덱스 블록에 모든 블록 번호 저장 | 직접 접근 가능, 외부 단편화 없음 | 인덱스 블록 오버헤드 |

#### 디렉터리 구조
| 구조 | 특징 |
|------|------|
| 1단계 | 모든 파일이 하나의 디렉터리. 이름 충돌 |
| 2단계 | 사용자별 디렉터리. 사용자 간 격리 |
| 트리 | 계층적 구조. 현대 OS에서 사용 |
| 비순환 그래프 | 파일 공유 가능. 링크 사용 |
| 일반 그래프 | 순환 허용. 가비지 컬렉션 필요 |

### 3.6 세마포어와 고전적 동기화 문제

#### 생산자-소비자 문제 (Bounded Buffer)
- 공유 버퍼에 생산자가 데이터를 넣고, 소비자가 데이터를 꺼냄
- 세마포어 3개 필요: mutex(1), full(0), empty(n)

#### 독자-저자 문제 (Readers-Writers)
- 여러 독자는 동시 읽기 가능
- 저자는 배타적 접근 필요 (읽기/쓰기 동시 불가)

#### 식사하는 철학자 문제 (Dining Philosophers)
- 5명의 철학자, 5개의 포크
- 교착 상태 발생 가능
- 해결: 비대칭 해법, 최대 4명만 동시 앉기, 두 포크 동시 집기 등

### [운영체제 예상 문제]

**문제 1.** 프로세스 상태 전이에서, 실행(Running) 상태에서 준비(Ready) 상태로 전이되는 원인은?
1. I/O 요청
2. 타임 슬라이스 만료
3. 프로세스 생성
4. I/O 완료

> **정답: 2번**. 시간 할당량(타임 슬라이스)이 만료되면 타이머 인터럽트에 의해 실행→준비 상태로 전이됩니다.

**문제 2.** 다음 중 프로세스 간 통신(IPC) 방법 중 가장 빠른 것은?
1. 파이프
2. 메시지 큐
3. 공유 메모리
4. 소켓

> **정답: 3번**. 공유 메모리는 커널을 거치지 않고 직접 메모리에 접근하므로 가장 빠릅니다.

**문제 3.** 페이징 기법에서 발생할 수 있는 단편화 종류는?
1. 외부 단편화만
2. 내부 단편화만
3. 외부 단편화와 내부 단편화 모두
4. 단편화가 발생하지 않음

> **정답: 2번**. 페이징은 고정 크기로 분할하므로 외부 단편화는 없지만, 마지막 페이지에서 내부 단편화가 발생할 수 있습니다.

**문제 4.** 다음 메모리 할당 기법 중, 요청 크기보다 큰 공간 중 가장 작은 것을 선택하는 방식은?
1. 최초 적합 (First Fit)
2. 최적 적합 (Best Fit)
3. 최악 적합 (Worst Fit)
4. 순환 적합 (Next Fit)

> **정답: 2번**

**문제 5.** 스레드에 대한 설명으로 틀린 것은?
1. 같은 프로세스의 스레드는 코드, 데이터, 힙 영역을 공유한다
2. 각 스레드는 자신만의 스택과 프로그램 카운터를 가진다
3. 스레드 간 문맥 교환은 프로세스 간 문맥 교환보다 오버헤드가 크다
4. 하나의 프로세스 내에서 여러 스레드가 동시에 실행될 수 있다

> **정답: 3번**. 스레드 간 문맥 교환은 프로세스 간보다 오버헤드가 **작습니다**. 주소 공간을 공유하므로 TLB 플러시 등이 필요 없습니다.

**문제 6.** 교착 상태의 필요조건 4가지 중, "프로세스가 자원을 점유한 채 다른 자원을 기다리는 것"에 해당하는 조건은?
1. 상호 배제
2. 점유와 대기
3. 비선점
4. 순환 대기

> **정답: 2번**

**문제 7.** 벨라디의 이상 현상(Belady's Anomaly)이 발생할 수 있는 페이지 교체 알고리즘은?
1. LRU
2. Optimal
3. FIFO
4. LFU

> **정답: 3번**. FIFO에서는 프레임 수를 늘려도 페이지 폴트가 증가하는 벨라디 이상 현상이 발생할 수 있습니다. LRU와 Optimal은 스택 알고리즘이므로 이 현상이 발생하지 않습니다.

**문제 8.** 파일 할당 방법 중, 각 블록에 다음 블록의 포인터를 저장하는 방식의 단점은?
1. 외부 단편화가 발생한다
2. 직접(임의) 접근이 느리다
3. 파일 크기를 미리 알아야 한다
4. 인덱스 블록의 공간 낭비가 크다

> **정답: 2번**. 연결 할당은 순차 접근은 가능하지만, 특정 블록에 직접 접근하려면 처음부터 링크를 따라가야 하므로 느립니다.

### 4.1 관계 대수 (Relational Algebra)

| 연산 | 기호 | 설명 |
|------|------|------|
| **셀렉션 (Selection)** | σ | 행(튜플) 선택. WHERE 조건 |
| **프로젝션 (Projection)** | π | 열(속성) 선택. SELECT 컬럼 |
| **합집합 (Union)** | ∪ | 두 릴레이션의 합 (중복 제거) |
| **차집합 (Difference)** | - | 한쪽에만 있는 튜플 |
| **교집합 (Intersection)** | ∩ | 양쪽 모두에 있는 튜플 |
| **카티션 곱 (Cartesian Product)** | × | 모든 조합 |
| **조인 (Join)** | ⋈ | 조건을 만족하는 조합 |
| **디비전 (Division)** | ÷ | "모두 포함"하는 튜플 선택 |

### 4.2 키 (Key) 종류

| 키 | 설명 |
|------|------|
| **슈퍼키** | 유일성을 만족하는 속성의 집합 (최소성 불필요) |
| **후보키** | 유일성 + 최소성을 모두 만족 |
| **기본키 (Primary Key)** | 후보키 중 선택된 하나. NULL 불가 |
| **대체키 (Alternate Key)** | 기본키로 선택되지 않은 후보키 |
| **외래키 (Foreign Key)** | 다른 릴레이션의 기본키를 참조 |

### 4.3 무결성 제약조건

| 종류 | 설명 |
|------|------|
| **개체 무결성** | 기본키는 NULL이 될 수 없음 |
| **참조 무결성** | 외래키 값은 참조하는 릴레이션의 기본키에 존재해야 함 (또는 NULL) |
| **도메인 무결성** | 속성값은 정의된 도메인에 속해야 함 |
| **키 무결성** | 릴레이션에는 최소한 하나의 키가 존재해야 함 |

#### 참조 무결성 위반 시 처리 옵션
| 옵션 | 설명 |
|------|------|
| CASCADE | 참조하는 튜플도 함께 삭제/수정 |
| SET NULL | 외래키를 NULL로 설정 |
| SET DEFAULT | 외래키를 기본값으로 설정 |
| RESTRICT | 삭제/수정 거부 (NO ACTION과 유사) |

### 4.4 SQL 고급 문법

#### 서브쿼리 종류
| 종류 | 위치 | 설명 |
|------|------|------|
| **스칼라 서브쿼리** | SELECT | 단일 값 반환 |
| **인라인 뷰** | FROM | 임시 테이블 역할 |
| **중첩 서브쿼리** | WHERE | 조건에 사용 |

#### 서브쿼리 연산자
| 연산자 | 설명 |
|--------|------|
| **IN** | 목록에 포함되는지 |
| **EXISTS** | 서브쿼리 결과가 존재하는지 (상관 서브쿼리에 적합) |
| **ANY/SOME** | 하나라도 만족하면 TRUE |
| **ALL** | 모두 만족해야 TRUE |

#### 윈도우 함수
```sql
SELECT name, dept, salary,
    RANK() OVER (PARTITION BY dept ORDER BY salary DESC) as rank,
    ROW_NUMBER() OVER (ORDER BY salary DESC) as row_num,
    SUM(salary) OVER (PARTITION BY dept) as dept_total
FROM employees;
```

| 함수 | 설명 |
|------|------|
| ROW_NUMBER() | 순위 (동일 값에도 다른 번호) |
| RANK() | 순위 (동일 값에 같은 번호, 다음 건너뜀: 1,2,2,4) |
| DENSE_RANK() | 순위 (동일 값에 같은 번호, 다음 이어감: 1,2,2,3) |
| LAG() / LEAD() | 이전/다음 행의 값 참조 |
| SUM() OVER() | 누적/그룹 합계 |

### 4.5 뷰 (View)

| 특징 | 설명 |
|------|------|
| 정의 | 하나 이상의 테이블로부터 유도된 가상 테이블 |
| 저장 | 데이터 자체가 아닌 **정의(쿼리)만 저장** |
| 장점 | 논리적 독립성, 보안(접근 제한), 편의성 |
| 갱신 제한 | 집계함수, DISTINCT, GROUP BY, JOIN이 포함된 뷰는 갱신 불가 |

### 4.6 동시성 제어 (Concurrency Control)

#### 로킹 (Locking)
| 잠금 종류 | 설명 |
|----------|------|
| **공유 잠금 (S-lock)** | 읽기 허용. 다른 트랜잭션도 S-lock 가능 |
| **배타 잠금 (X-lock)** | 읽기/쓰기 모두 독점. 다른 잠금 불허 |

#### 2단계 잠금 규약 (2PL: Two-Phase Locking)
- **확장 단계(Growing Phase)**: 잠금만 획득, 해제 불가
- **축소 단계(Shrinking Phase)**: 잠금만 해제, 획득 불가
- 직렬 가능성을 보장하지만 교착 상태 발생 가능

#### 타임스탬프 순서 규약
- 트랜잭션에 타임스탬프를 부여하여 순서 결정
- 잠금 없이 동시성 제어 → 교착 상태 없음
- 연쇄 복귀(Cascading Rollback) 발생 가능

### 4.7 분산 데이터베이스

#### 분산 데이터베이스의 투명성
| 투명성 | 설명 |
|--------|------|
| **위치 투명성** | 데이터의 물리적 위치를 몰라도 접근 가능 |
| **분할 투명성** | 데이터의 분할 여부를 몰라도 사용 가능 |
| **복제 투명성** | 데이터의 복제 여부를 몰라도 사용 가능 |
| **장애 투명성** | 일부 장애 시에도 정상 동작 |
| **병행 투명성** | 동시 접근 시 결과의 일관성 보장 |

### 4.8 E-R 모델

| 구성 요소 | 기호 | 설명 |
|----------|------|------|
| 개체 (Entity) | 사각형 | 실세계의 객체 (학생, 과목) |
| 속성 (Attribute) | 타원 | 개체의 특성 (이름, 학번) |
| 관계 (Relationship) | 마름모 | 개체 간 연관 (수강) |
| 키 속성 | 밑줄 타원 | 개체를 유일하게 식별 |
| 다중값 속성 | 이중 타원 | 하나의 속성에 여러 값 (전화번호) |
| 유도 속성 | 점선 타원 | 다른 속성에서 유도 (나이 = 현재-생년) |
| 약한 개체 | 이중 사각형 | 자체 키가 없어 다른 개체에 의존 |

#### 카디널리티 (대응 수)
- **1:1** — 하나의 개체가 다른 하나의 개체와 대응
- **1:N** — 하나의 개체가 여러 개체와 대응
- **M:N** — 여러 개체가 여러 개체와 대응 (중간 테이블 필요)

### 5.1 스택 활용

| 활용 | 설명 |
|------|------|
| 함수 호출 관리 | 콜 스택 (지역 변수, 복귀 주소 저장) |
| 수식 변환 | 중위→후위(RPN) 변환 |
| 수식 계산 | 후위 표기법 계산 |
| 괄호 검사 | 여는 괄호 push, 닫는 괄호 pop하여 매칭 |
| 실행 취소 (Undo) | 최근 작업부터 역순으로 취소 |
| DFS | 깊이 우선 탐색 |
| 역순 문자열 | 문자를 push 후 pop |

#### 후위 표기법 변환 (중위→후위)
중위: A * (B + C) → 후위: ABC+*

알고리즘:
1. 피연산자 → 바로 출력
2. 여는 괄호 → 스택에 push
3. 닫는 괄호 → 여는 괄호가 나올 때까지 pop하여 출력
4. 연산자 → 스택 top보다 우선순위가 높으면 push, 아니면 pop 후 push
5. 수식 끝 → 스택의 모든 연산자 pop하여 출력

### 5.2 큐 활용과 변형

| 종류 | 설명 |
|------|------|
| **선형 큐** | FIFO. front와 rear로 관리 |
| **원형 큐** | 배열의 끝과 처음이 연결. 공간 낭비 해결. 공백/포화 구분 위해 한 칸 비움 |
| **우선순위 큐** | 우선순위가 높은 것이 먼저 나옴. 힙으로 구현 |
| **덱 (Deque)** | 양쪽에서 삽입/삭제 가능 |

#### 원형 큐 계산
- 공백 조건: front == rear
- 포화 조건: (rear + 1) % SIZE == front
- 삽입: rear = (rear + 1) % SIZE
- 삭제: front = (front + 1) % SIZE

### 5.3 이진 탐색 트리 (BST)

| 특성 | 설명 |
|------|------|
| 왼쪽 서브트리 | 루트보다 작은 값 |
| 오른쪽 서브트리 | 루트보다 큰 값 |
| 중위 순회 | 정렬된 순서로 출력 |
| 평균 탐색 | O(log n) |
| 최악 탐색 | O(n) — 편향 트리 |

#### BST 삭제 연산
| 경우 | 처리 |
|------|------|
| 리프 노드 | 그냥 삭제 |
| 자식 1개 | 자식을 부모에 연결 |
| 자식 2개 | 중위 후속자(오른쪽 서브트리의 최솟값) 또는 중위 선행자(왼쪽 서브트리의 최댓값)로 대체 |

### 5.4 AVL 트리

- 모든 노드에서 왼쪽/오른쪽 서브트리 높이 차이가 1 이하
- 균형이 깨지면 회전 연산으로 복구
- 탐색/삽입/삭제 모두 O(log n) 보장

#### 회전 유형
| 유형 | 발생 조건 | 해결 |
|------|----------|------|
| LL | 왼쪽 자식의 왼쪽에 삽입 | 오른쪽 회전 1회 |
| RR | 오른쪽 자식의 오른쪽에 삽입 | 왼쪽 회전 1회 |
| LR | 왼쪽 자식의 오른쪽에 삽입 | 왼쪽 회전 후 오른쪽 회전 |
| RL | 오른쪽 자식의 왼쪽에 삽입 | 오른쪽 회전 후 왼쪽 회전 |

### 5.5 B-트리 / B+트리

| 구분 | B-트리 | B+트리 |
|------|--------|--------|
| 데이터 위치 | 모든 노드 | 리프 노드만 |
| 리프 노드 연결 | 연결 없음 | 연결 리스트로 연결 |
| 순차 접근 | 비효율적 | 효율적 (리프 순회) |
| 키 중복 | 없음 | 내부 노드에 키 복사 |
| 데이터베이스 인덱스 | 사용 | **주로 사용** |

### 5.6 최소 신장 트리 (MST)

| 알고리즘 | 방식 | 시간 복잡도 |
|----------|------|-----------|
| **크루스칼 (Kruskal)** | 간선 중심. 가중치 오름차순 정렬 후 사이클 없이 선택 | O(E log E) |
| **프림 (Prim)** | 정점 중심. 시작 정점에서 인접 간선 중 최소 선택 | O(E log V) |

- 크루스칼: Union-Find로 사이클 판별. 간선이 적을 때 유리 (희소 그래프)
- 프림: 우선순위 큐 사용. 간선이 많을 때 유리 (밀집 그래프)

### 5.7 위상 정렬 (Topological Sort)

- DAG(Directed Acyclic Graph)에서만 가능
- 진입 차수가 0인 정점부터 제거하며 순서를 결정
- 선행 조건이 있는 작업 스케줄링에 사용
- 사이클이 있으면 위상 정렬 불가

### 5.8 시간 복잡도 정리

| 표기법 | 의미 |
|--------|------|
| O(1) | 상수 시간 |
| O(log n) | 로그 시간 (이진 탐색) |
| O(n) | 선형 시간 (순차 탐색) |
| O(n log n) | 로그 선형 시간 (효율적 정렬) |
| O(n²) | 이차 시간 (버블, 선택, 삽입 정렬) |
| O(2ⁿ) | 지수 시간 (부분 집합 문제) |
| O(n!) | 팩토리얼 시간 (순열 문제) |

### 6.1 요구사항 분석

### 6.2 SOLID 원칙

| 원칙 | 이름 | 설명 |
|------|------|------|

| **O** | 개방-폐쇄 원칙 (OCP) | 확장에는 열려있고, 수정에는 닫혀있어야 함 |
| **L** | 리스코프 치환 원칙 (LSP) | 자식 클래스는 부모 클래스를 대체할 수 있어야 함 |

| **D** | 의존성 역전 원칙 (DIP) | 고수준 모듈이 저수준 모듈에 의존하면 안 됨. 추상화에 의존 |

### 6.3 테스트 수준

| 수준 | 설명 | 수행 주체 |
|------|------|----------|
| **단위 테스트** | 개별 모듈/함수 테스트 | 개발자 |
| **통합 테스트** | 모듈 간 인터페이스 테스트 | 개발팀 |
| **시스템 테스트** | 전체 시스템 요구사항 검증 | QA팀 |
| **인수 테스트** | 사용자 관점에서 시스템 검증 | 사용자/고객 |

#### 통합 테스트 전략
| 전략 | 설명 |
|------|------|
| **빅뱅** | 모든 모듈을 한꺼번에 통합. 오류 위치 파악 어려움 |
| **하향식 (Top-down)** | 상위 모듈부터 통합. 스텁(Stub) 사용 |
| **상향식 (Bottom-up)** | 하위 모듈부터 통합. 드라이버(Driver) 사용 |
| **샌드위치** | 하향식 + 상향식 결합 |

### 6.4 소프트웨어 품질 특성 (ISO/IEC 25010)

| 특성 | 설명 |
|------|------|
| **기능적합성** | 요구 기능을 제공하는 정도 |
| **성능 효율성** | 자원 사용량 대비 성능 수준 |
| **호환성** | 다른 시스템과의 공존/상호운용 능력 |
| **사용성** | 사용의 용이성 |
| **신뢰성** | 정상 동작을 유지하는 능력 (성숙성, 가용성, 결함허용, 복구성) |
| **보안성** | 정보 보호 능력 |
| **유지보수성** | 수정의 용이성 (모듈성, 재사용성, 분석성, 시험가능성) |
| **이식성** | 다른 환경으로 이전 능력 |

### 6.5 형상 관리 (Configuration Management)

| 활동 | 설명 |
|------|------|
| **형상 식별** | 관리 대상 항목(소스코드, 문서 등) 선정 |
| **형상 통제** | 변경 요청의 승인/거부 (CCB: 형상 통제 위원회) |
| **형상 감사** | 변경이 올바르게 수행되었는지 확인 |
| **형상 기록/보고** | 변경 이력 기록 및 상태 보고 |

### 6.6 프로젝트 일정 관리

#### PERT/CPM
| 용어 | 설명 |
|------|------|

| **EF (Earliest Finish)** | 가장 빠른 완료 시간 |
| **LS (Latest Start)** | 가장 늦은 시작 시간 |
| **LF (Latest Finish)** | 가장 늦은 완료 시간 |

### 6.8 객체지향 개념

| 개념 | 설명 |
|------|------|
| **캡슐화** | 데이터와 메서드를 하나로 묶고 정보를 은닉 |
| **상속** | 부모 클래스의 속성/메서드를 자식이 물려받음 |
| **다형성** | 같은 이름의 메서드가 다르게 동작 (오버로딩, 오버라이딩) |
| **추상화** | 핵심적인 부분만 모델링하고 세부사항 숨김 |

#### 오버로딩 vs 오버라이딩
| 구분 | 오버로딩 (Overloading) | 오버라이딩 (Overriding) |
|------|----------------------|----------------------|
| 정의 | 같은 이름, 다른 매개변수 | 같은 이름, 같은 매개변수 |
| 위치 | 같은 클래스 내 | 부모-자식 클래스 간 |
| 바인딩 | 컴파일 타임 (정적) | 런타임 (동적) |
| 목적 | 편의성 (같은 기능, 다른 입력) | 재정의 (같은 메서드, 다른 동작) |

### [소프트웨어 공학 예상 문제]

**문제 1.** 통합 테스트에서 상위 모듈을 먼저 테스트하고, 하위 모듈을 점진적으로 통합할 때 필요한 보조 모듈은?
1. 드라이버 (Driver)
2. 스텁 (Stub)
3. 오라클 (Oracle)
4. 모의 객체 (Mock)

> **정답: 2번**. 하향식 통합 테스트에서는 아직 개발되지 않은 하위 모듈 대신 **스텁(Stub)**을 사용합니다. 반대로 상향식은 **드라이버(Driver)**를 사용합니다.

**문제 2.** SOLID 원칙 중 "확장에는 열려있고, 수정에는 닫혀있어야 한다"는 원칙은?
1. 단일 책임 원칙
2. 개방-폐쇄 원칙
3. 리스코프 치환 원칙
4. 의존성 역전 원칙

> **정답: 2번**. OCP(Open-Closed Principle)

**문제 3.** 스크럼에서 제품의 요구사항 목록을 관리하고 우선순위를 결정하는 역할은?
1. 스크럼 마스터
2. 제품 소유자
3. 개발팀
4. 이해관계자

> **정답: 2번**

**문제 4.** 다음 중 비기능적 요구사항에 해당하는 것은?
1. 사용자 로그인 기능
2. 상품 주문 처리 기능
3. 시스템 응답시간 2초 이내
4. 장바구니 담기 기능

> **정답: 3번**. 성능에 관한 것은 비기능적 요구사항입니다.

**문제 5.** 임계 경로(Critical Path)에 대한 설명으로 옳은 것은?
1. 프로젝트에서 가장 비용이 많이 드는 경로이다
2. 여유 시간이 가장 많은 경로이다
3. 프로젝트 완료에 가장 긴 시간이 걸리는 경로이다
4. 가장 적은 수의 작업으로 구성된 경로이다

> **정답: 3번**. 임계 경로의 소요 시간 = 프로젝트의 최소 완료 시간. 임계 경로의 여유 시간은 0입니다.

**문제 6.** 형상 관리에서 변경 요청에 대한 승인 여부를 결정하는 조직은?
1. 개발팀
2. QA팀
3. 형상 통제 위원회 (CCB)
4. 프로젝트 관리자

> **정답: 3번**

**문제 7.** 객체지향에서 "같은 이름의 메서드가 매개변수의 타입이나 개수에 따라 다르게 동작하는 것"을 무엇이라 하는가?
1. 오버라이딩
2. 오버로딩
3. 캡슐화
4. 추상화

> **정답: 2번**. 오버로딩(Overloading)은 같은 이름, 다른 매개변수입니다. 오버라이딩(Overriding)은 부모 클래스의 메서드를 자식 클래스에서 재정의하는 것입니다.

**문제 8.** XP(eXtreme Programming)의 실천법이 아닌 것은?
1. 페어 프로그래밍
2. 스프린트 계획 미팅
3. TDD (테스트 주도 개발)
4. 리팩토링

> **정답: 2번**. 스프린트 계획 미팅은 **스크럼**의 활동입니다. XP의 실천법에는 페어 프로그래밍, TDD, 리팩토링, 지속적 통합, 작은 릴리즈 등이 있습니다.

### 헷갈리기 쉬운 쌍

| 개념 A | 개념 B | 핵심 차이 |
|--------|--------|----------|
| 프로세스 | 스레드 | 프로세스는 독립 메모리, 스레드는 메모리 공유 |
| 세마포어 | 뮤텍스 | 세마포어는 카운팅 가능, 뮤텍스는 소유권 개념 |
| 교착 상태 | 기아 상태 | 교착은 서로 대기, 기아는 무한 대기 |
| 페이징 | 세그멘테이션 | 페이징은 고정 크기, 세그먼트는 가변 크기 |
| TCP | UDP | TCP는 연결 지향+신뢰성, UDP는 비연결+비신뢰성 |
| 허브 | 스위치 | 허브는 모든 포트에 전달, 스위치는 목적지 포트만 |
| 라우터 | 스위치 | 라우터는 3계층(IP), 스위치는 2계층(MAC) |
| CSMA/CD | CSMA/CA | CD는 충돌 감지(유선), CA는 충돌 회피(무선) |
| ARP | RARP | ARP는 IP→MAC, RARP는 MAC→IP |
| 크루스칼 | 프림 | 크루스칼은 간선 중심, 프림은 정점 중심 |
| BFS | DFS | BFS는 큐(최단 경로), DFS는 스택(백트래킹) |
| 스텁 | 드라이버 | 스텁은 하향식(하위 대체), 드라이버는 상향식(상위 대체) |
| 오버로딩 | 오버라이딩 | 오버로딩은 같은 클래스 내 다른 매개변수, 오버라이딩은 상속 관계에서 재정의 |
| 하드 링크 | 심볼릭 링크 | 하드는 inode 공유, 심볼릭은 경로 저장 |
| 기본키 | 외래키 | 기본키는 자신 테이블 식별, 외래키는 다른 테이블 참조 |
| 뷰 | 테이블 | 뷰는 정의만 저장, 테이블은 실제 데이터 저장 |
| Redo | Undo | Redo는 완료된 것 재실행, Undo는 미완료 것 되돌림 |
